# bossy

Adversarial productivity boss that monitors your idle time and periodically checks in to keep you on task.

## What it does

bossy is an adversarial productivity boss that monitors idle time, escalates check-ins (first after 30 min, then every 10 min, up to 3), injects boss-voice turns via custom messages, and shows a persistent status line. Uses the existing `todo` tool for task state.

## Prerequisites

- pi installed and working
- The `todo` extension loaded (bossy scans session for `todo` tool results)

## Installation (dev / local)

```bash
# Point pi directly at the source file (no install needed)
pi -e C:/repo/pi-extensions/bossy/src/index.ts

# Or symlink to pi's global extensions folder
ln -s C:/repo/pi-extensions/bossy ~/.pi/agent/extensions/bossy
```

## Installation (production / pi install)

```bash
cd C:/repo/pi-extensions/bossy
pi install .
```

Load both extensions in `settings.json`:

```json
{
  "extensions": [
    "~/.pi/agent/extensions/todo/todo.ts",
    "C:/repo/pi-extensions/bossy/src/index.ts"
  ]
}
```

## Commands

- `/boss` — open the boss panel (tasks, idle time, check-in now)
- `/boss on` / `/boss off` — toggle the boss
- `/boss checkin` — manually trigger a check-in
- `/boss config` — edit thresholds (idle, follow-up interval, max follow-ups)
- `/boss help` — show this list

## Configuration

Env vars for testing / CI:

- `BOSSY_IDLE_MS` — ms before first ping (default: 1800000)
- `BOSSY_FOLLOWUP_MS` — ms between follow-ups (default: 600000)
- `BOSSY_MAX_FOLLOWUPS` — max follow-ups per idle period (default: 3)
- `BOSSY_DISABLED=1` — start with boss disabled

Example: `BOSSY_IDLE_MS=60000 pi -e ./src/index.ts` fires a check-in after 60 seconds.

## How the boss decides to check in

Rule B only: boss fires only when there is at least one open todo. If the `todo` extension isn't loaded or there are no open tasks, boss stays quiet even after 30 min of idle.

## How to tune the check-in language

The boss's actual wording ("Status." vs "Working?") is generated by the LLM at runtime. To change the tone, edit `src/personality.ts` — the system prompt injection controls the LLM's behavior.
